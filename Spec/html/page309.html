CHAPTER 29. PANES

These spacing options apply to hbox-pane, vbox-pane, table-pane, and grid-pane, and indi-
cate the amount of horizontal and vertical spacing (respectively) to leave between the items in
boxes or rows and columns in table. The values of these options are specified in the same way

29.3.2 Layout Pane Classes

=> hbox-pane [Layout Pane]
=> horizontally (&rest options) &body contents [Macro]
The hbox-pane class lays out all of its child panes horizontally, from left to right. The horizontally
macro serves as a convenient interface for creating an hbox-pane.
contents is one or more forms that are the child panes. Each form in contents is of the form:
  • A pane. The pane is inserted at this point and its space requirements are used to compute
    the size.

  • A number. The specified number of device units should be allocated at this point.

  • The symbol +fill+. This means that an arbitrary amount of space can be absorbed at
    this point in the layout.

  • A list whose first element is a number and whose second element evaluates to a pane. If
    the number is less than 1, then it means that that percentage of excess space or deficit
    should be allocated to the pane. If the number is greater than or equal to 1, then that
    many device units are allocated to the pane. For example:
     
 (horizontally ()
   (1/3 (make-pane 'label-button-pane))
   (2/3 (make-pane 'label-button-pane)))
    would create a horizontal stack of two button panes. The first button takes one-third of
    the space, then second takes two-thirds of the space.
=> vbox-pane [Layout Pane]
=> vertically (&rest options) &body contents [Macro]
The vbox-pane class lays out all of its child panes vertically, from top to bottom. The vertically
macro serves as a convenient interface for creating an vbox-pane.

contents is as for horizontally.
